Fase Actividades
Iniciacion
  • Nace la idea del producto de software.
  • Se identifican los clientes y segmentos del mercado.
  • Se entrevistan algunos clientes para identificar sus necesidades y conocer de forma general las funcionalidades que debería tener el SW.
  • Se hace un análisis para determinar si existen otros productos similares.
  • Se analiza la factibilidad del producto y la posible aceptación por parte de los usuarios.
Planeacion
  • Se desarrolla un plan para determinar los recursos necesarios para implementar el proyecto de software (tiempos, cronograma, personal, etc).
  • Se determina el costo aproximado del proyecto y el tiempo de desarrollo.
  • Dependiendo de la fecha de lanzamiento del producto se puede ajustar el cronograma.
  • El plan de desarrollo del producto se actualiza de acuerdo con las necesidades del proyecto (ej: si se incluyen nuevas funcionalidades)
  • Se desarrolla el Plan de Gestión de la Configuración: Contiene actividades para controlar los cambios y manejar versiones de los diseños.
Analisis
  • Se recopila la mayor cantidad de información posible sobre las necesidades de los clientes .
  • Se recopila información detallada sobre las funcionalidades del producto, desempeño requerido para el producto, confiabilidad, seguridad, usabilidad.
  • Los requerimientos generales se pueden recopilar por medio de entrevistas y lluvias de ideas con los clientes.
  • Los requerimientos definen qué es lo que el producto de software debe hacer.
Diseño
  • Esta fase define cómo el software debe ser construido para cumplir con los requerimientos definidos en la fase anterior.
  • Se define la arquitectura del software (división en sub-sistemas) y se diseña de forma específica cada componente del software. Todo esto se acompaña de la respectiva documentación que es lo que conforma el documento de diseño del software.
  • Nota: Hasta este punto no se ha programado nada. No se ha desarrollado código aún.
    •
Implementacion
  • En esta fase se programa todo el software.
  • El objetivo es tomar el diseño obtenido en la fase anterior y programarlo completamente usando las tecnologías que se hayan definido.
  • Como parte de la programación del software, se integran otros componentes de software como componentes de otros fabricantes.
  • El resultado de esta fase es el código del software listo para ser probado.
Pruebas
  • El software se prueba para determinar si cumple con los requerimientos
  • El software se prueba en tres fases:
    • Cada módulo se prueba de forma independiente.
    • Los módulos se van integrando para determinar si la comunicación entre los módulos es correcta.
    • Una vez se integran todos los módulos y se conforma el sistema completo, se determina si cumple con los requerimientos.
  • Una vez se completan estas pruebas, se ejecutan varias pruebas con los clientes y se pasa a «beta testing»
Mantenimiento
  • Esta fase comienza cuando el software se libera oficialmente para los clientes.
  • En esta fase:
    • Se reparan fallos que se presenten durante el uso por parte de los clientes.
    • Se implementan requerimientos adicionales de los clientes con respecto a mejoras.
    • Se mejoran atributos del SW como confiabilidad, desempeño, etc.
  • Después de cada mantenimiento se genera una nueva versión del software.
