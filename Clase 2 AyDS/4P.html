En un proyecto de software participan diferentes equipos:
  1. Gestión del Negocio: Personas encargadas de la compañía que desarrolla el software (Personas de finanzas, Marketing, Gestión del producto, etc)
  2. Gestión del Proyecto: Personas encargadas de la planificación del proyecto. Se encargan de gestionar las personas, los procesos y las actividades del proyecto y monitorean el proyecto para mantenerlo dentro del cronograma y dentro del presupuesto.
  3. Equipo de Desarrollo: Son los ingenieros de software y desarrolladores encargados de recopilar requerimientos, diseñar la arquitectura, implementar el software, ejecutar pruebas y documentar.
    4.

El producto es el Software como tal. Sin embargo el producto debe ir acompañado de:
  1. Documentación del proyecto
  2. Código fuente
  3. Documentos de pruebas
  4. Documentos de cliente (Manuales de usuario, de instalación, etc.)
    5.

Un proyecto de software comprende las actividades necesarias para generar un producto de software. Las fases que sigue un proyecto de software generalmente son:
  1. Planeacion
  2. Analisis y requerimientos
  3. Diseño
  4. Implementacion
  5. Pruebas
  6. Mantenimineto
    7.

Un modelo de proceso de software es la manera de organizar de una forma sistemática las personas y actividades para el desarrollo de un producto de software en el marco de un proyecto de software.
